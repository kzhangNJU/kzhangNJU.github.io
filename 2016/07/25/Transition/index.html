转场动画概述

自定义转场动画

Transition Animation Sequence

一个专场动画其实就是交换两个controller的内容。有两种类型的转换:presentation和dismissal。Presentation转换向你的APP的视图结构中增加一个新的controller，dismissal 则是从结构中移除一个或多个controller。

需要多个对象才能实现一个专场动画。UIKit提供了这个些转场需要的对象的默认版本，你可以自定义其中的一些或者全部。如果你选择了正确的自定义集合，那么只需要很少的代码量就可以实现你想要的效果。如果利用好了UIKit提供的现有代码，就算事包含交互的动画也是很容易实现的。

Transitioning Delegate （转场代理）

转场代理事转场动画和自定义展示的起点。转场代理是你自定义的对象，需要服从UIViewControllerTransitioningDelegate协议。它的工作就是提供UIKit下列对象：

  • Animator objects. 一个animator对象负责创建展示或隐藏controller所需要的动画。转场代理能够提供不同的animator对象，animator对象要服从UIViewControllerAnimatedTransitioning协议。

  • Interactive animator objects.交互式的animator对象使用点击事件或者收拾来驱动转场的时机。交互式的animator对象要服从UIViewControllerInteractiveTransitioning协议。

    创建一个交互式的animator对象最简单的方式就是继承UIPercentDrivenInteractiveTransition类，向子类中添加事件处理的代码。这个类能够控制动画的时机。如果你创建自己的交互式animator，你必须自己渲染动画的每一帧。

  • Presentation Controller.Presentation Controller管理controller在屏幕上的显示风格。

给一个view controller的transitioningDelegate属性赋给你的专场代理，这样UIKit就知道你想实现自定义的转场和展示。你的代理对象能够选择提供哪些对象如果你不提供animator对象，UIKit使用modalTransitionStyle中的标准的转场动画。

The Custom Animation Sequence

当正在的展示的view Controller的transitioningDelegate属性包含一个有效对象时，UIKit使用你提供的自定义的animator对象来展示controller。当准备执行展示时，UIKit调用animationControllerForPresentedController:presentingController:sourceController:方法来获取自定义的animator对象。如果这个对象有效，UIKit会执行下面的步骤:

  1. UIKit调用转场代理的interactionControllerForPresention:方法来查看是否有有效的interactive animator对象。如果这个方法返回nil，UIKit执行动画时不使用用户交互。
  2. UIKit调用animator对象的transitionDuration:方法来获得动画持续时间。

  3. UIKit调用合适的方法来开始动画:

    • 对于非交互式的动画，UIKit调用animator对象的animateTranstion:方法。
    • 对于交互式动画，UIKit调用交互式animator的startInteractiveTransition:方法。

  4. UIKit等待animator对象调用completeTransiton:方法。自定义的animator会在动画结束时调用这个方法，一般是在动画结束的回调中。调用这方法来结束转场，告诉UIKit可以执行presentViewController:animated:completion:的完成回调和animao］tor的animationEnded:方法。

当关闭一个view controller时，UIKit调用animationControllerForDismissedController:方法，执行下面的步骤：

  1. UIKit调用转场代理的interactionCOntrollerForDismissal:方法查看有没有可用的interacive animator。如果这个方法返回nil，UIKit执行不带用户交互的动画。
  2. UIKit调用transitionDuration:方法获得动画时间。
  3. UIKit调用合适的方法来开启动画：
    • 对于非交互动画，UIKit调用animateTransition:方法
    • 对于交互式动画，UIKit调用startInteractiveTransition:方法。
  4. UIKit等待animator对象调用completeTransiton:方法。
    自定义的animator会在动画结束时调用这个方法，一般是在动画结束的回调中。调用这方法来结束转场，告诉UIKit可以执行presentViewController:animated:completion:的完成回调和animao］tor的animationEnded:方法。

　注意

在动画结束后调用completeTransition:方法是必须的。只有调用这个方法，UIKit才会结束转场过程，将APP的控制归还。

The Transitioning Context Object

在转场动画开始前，UIKit会创建一个转换上下文对象，将如何执行动画的信息填写到这个对象中。转场上下文对象是你代码中的重要一部分。它实现了UIViewControllerContextTranstitioning协议，存储转换过程中controller和view的引用。它也存储了如果执行转场的信息，包括动画是不是可交互的。

注意

当设置自定义动画时，使用转场上下文对象中存储的对象和数据，而不是你自己管理的缓存的信息。转场可能在各种情况下发生，其中某些情况会改变动画的参数。转场的上下文对象保障你需要的转场信息是正确的，而你自己缓存的信息可能是过时的。

The Transition Coordinator

不管是内置的转场还是自定义的转场，UIKit都会创建一个transition coordinator对象来管理你需要的额外的动画信息。除了presentation和dismissal，转场还可能发生在interface rotation或frame改变时。这些转场都代表着view继承结构的变化。transition coordinator用来追踪view继承结构的变化。一个transition coordinator只存活于转场过程中。

使用自定义动画展示视图控制器

步骤如下：

  1. 创建你要展示的controller
  2. 创建自定义的转场代理对象，将它赋给controller的transtitioningDelegate属性。
  3. 调用presentViewController:animated:completion:方法来展示view controller。

当你调用presentViewController:animated:completion时，UIKit会初始化展示过程。展示开始于下一个runloop开始，知道你自定义的animator调用completeTransition:。交互式的转场允许在转场发生时处理点击事件。

实现转场代理

转场代理的目的时创建自定义的对象。下面展示了最简单的转场方法的实现，这个例子创建了一个animator对象，大部分的工作是animator处理的。

实现你的Animator对象

Animator对象要满足UIViewControllerAnimatedTransitioning协议。Animator对象创建一个执行的动画。最关键的方法是animateTransition:，用它来创建实际的动画。动画处理过程大概可以分成下面几个部分：

  1. 获取动画参数
  2. 使用Core Animation或UIView动画方法来创建动画
  3. 清理和结束转场

获取动画参数

传递给animateTransition:的上下文转场对象包含了动画所需要的数据。永远不要食用你自己缓存的信息，或者从你的controller中获取信息。展示或关闭视图时可能会包含controller之外的对象。比如，一个自定义的展示控制器可能会添加一个背景颜色。上下文转场对象考略到了这些额外的视图和对象，让你进行正确的动画操作。

  • 调用两次viewControllerForKey:方法来获得转换过程中的from和to两个controller。永远不要假设你知道哪个controller参与转场。UIKit可能会修改controller。
  • 调用containerView方法来获得动画的superview。将所有的子视图添加到这个视图上。比如，在展示期间，将要展示的视图都加到这个视图上。
  • 调用viewForKey:方法来获得将被添加或移除的视图。Controller的视图可能不是唯一被添加或移除的视图。
  • 调用finalFrameForViewController:方法来获得view添加或移除的最终的帧。

创建转场动画

从转场上下文对象中获取数据来创建动画。

展示动画：

  • 使用viewControllerForKey:和viewForKey:方法来获取转场的view controller和view。
  • 设置要”to” view的起始位置。设置动画需要的初始值。
  • 调用上下文转场对象的finalFrameForViewController:方法获取”to” view的最终位置。
  • 将”to”view作为子view添加到容器view中
  • 创建动画
    • 在动画block中，将”to”view转换到最终位置。
    • 在完成block中，调用completeTransition:方法，进行清理工作。

动画结束后的清理

在转场动画结束后，调用completeTransition:方法是至关重要的。调用这个方法告诉UIKit转场已经结束，用户可以使用展示的controller了。调用这个方法也会出发一系列的其它回调，包括presentViewController:animated:completion:方法的回调和animator自己的animationEnded:方法。调用completeTransition:方法最好的地方就是动画的完成回调。

由于转场是可以取消的的，你应该使用转场上下文的transitionWasCancelled:方法来决定是否需要清理。当转场取消时，你的animator应该撤销所有的操作。

给转场添加交互性

给动画添加交互性的最简单的方式就是使用UIPercentDrivenInteractiveTransition对象。这个对象与现有的animator对象一起协同控制动画的时机。它通过你提供的完成百分比来工作。你所要做的就是设置事件处理代码，来计算当前转场完成的百分比，并进行更新。

你可以直接使用这个类或者继承它。如果继承，使用init方法（或startInteractiveTransition:）来进行一次性的事件处理初始化。然后，使用你自定义的事件处理代码计算完成百分比，调用updateInteracitveTransition:方法。当你的代码返回完成时，调用finishInteractiveTransition方法。

创建与转场同时进行的动画

转场中的view controller可以进行额外的动画。比如，一个要展示的controller可以在转场进行时进行自己视图的动画处理，给转场添加额外的视觉效果。任何对象都能够创建动画，只要它能访问transitionCoordinator属性。这个对象只在转场过程中存在。

要创建动画，调用animatedAlongsideTransistion:completion:或者animateAlongsideTransitionInView:animation:completion:方法。
