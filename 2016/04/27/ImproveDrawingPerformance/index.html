提升绘制性能

提升绘制性能

绘制是一个相对昂贵的开销，优化绘制的代码是开发过程中的重要步骤。

  • Draw minimally 每个更新循环，你应该只更新view改变的部分。如果你使用drawRect:方法，使用update rectangle来限制绘制区域。对于OpenGL绘制，你必须自己追踪更新。
  • 谨慎调用setNeedsDisplay: 如果你调用这个方法，不要只传入整个view的rectangle，只传入需要重新绘制的区域。
  • 尽量可能设置view为opague。组合opague的view开销会比透明的少很多。要让view不透明，view的内容必须不包含任何透明度，且view的opague属性为真。
  • 滑动过程中重用view和cell。滚动中应该尽可能避免创建新的view。创建新的view的花费会减少更新屏幕的可用时间，导致滚动不流畅。
  • 通过修改转换矩阵来重用path。通过修改当前转换矩阵，你可以只用一个path在屏幕的不同地方进行绘制
  • 滚动过程中避免清空前面的内容。默认情况下，UIKit更新一片区域前，会先清除当前的context buffer，然后再调用drawRect:。要禁止这种行为，可以将clearsContextBeforeDrawing属性设置为No。

Instruments的性能分析

Core Animation中的几个Debug选项的含义：

  • Color Blended Layers.显示混合的layers。多个view layers混合在一起时会被红色标亮。Blended view layers是引起table滚动卡顿的主要原因。
  • Color Hits Green and Misses Red。将views标记为绿色或红色，可以使用cached rasterization的view被标记为绿色。
  • Color Copied Images。
  • Color Misaligned Images。洋红色，源像素跟目标像素不匹配的情况。
  • Color offscreen－Rendered Yellow。
  • Flash updated regions。

Offscreen rendering／Rendering on the CPU

Offscreen rendering(softwear rendering)：在交给GPU之前，绘制需要让CPU先处理。硬件不会处理text rendering以及mask和shadow。下面的情况会导致Offscreen rendering:

  • Layer with a mask
  • layer的masksToBounds或view.clipsToBounds为真
  • layer的allowsGroupOpactiy和layer opacity少于1
  • 设置了layer的shadow
  • layer的shouldRasterize
  • Text
  • 大多数的drawRect中的CGContext

Blending

  • resizableImage会导致blending
  • opague为NO或背景alph不是1
  • layer.content或UIImageView的Image有alpha通道
