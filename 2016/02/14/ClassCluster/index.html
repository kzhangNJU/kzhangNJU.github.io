Class Cluster基本概念

Class Cluster

Class Cluster是这样是一个结构，它将一些私有的、具体的子类组合在一个公共的、抽象的superclass下。通过这种方式组织，能给用户提供一个简化的借口，用户只能看见这个结构的公共可见部分。抽象类会调用子类的私有类最适合的方法来执行一个特定的任务。

比如，许多Cocoa类都是Class Cluster，像NSArray、NSString和NSDictionary。内部的数据存储有很多中方式，对于特定的实例，抽象类会基于初始化实例的数据选择最高效的类。

Cluster Class的使用方式跟其他类没有什么区别。只是在实现上，当你创建公共类的实例时，这个类会基于你调用的创建方法返回适合的子类的实例。（你不能选择实例的实际类。）

好处

Class Cluster的主要好处在于效率。数据的内部表示可以根据数据创建或使用的方式来决定。而且，你的代码在内部实现修改后仍可以继续使用。

思考

Class Cluster权衡了便捷性和扩展性。用一些公共类来代替大量的私有类使得框架的学习和使用更简单，但是使得创建cluster内的类变得困难。

要创建一个cluster内的类必须这样做：

  • 必须是cluster的abstract superclass的子类
  • 声明自己的storage（自己需要的实例变量）
  • 重写superclass所有的初始化方法
  • 重写superclass的原生方法（直接访问类实例变量的方法）

你的子类应该声明自己的init方法（如果需要初始化实例变量）和+className方法。它不应该依靠任何继承的东西。为了保持初始化的链，它应该调用superclass类的初始化方法。
很少情况下需要创建cluster内的子类。你可以通过合成的方式来避免继承，将一个私有的cluster对象嵌入你自己设计的对象。
